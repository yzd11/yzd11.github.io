Stream API vs 集合框架：

Stream API 关注的是多个数据之间的计算（排序，查找，过滤，映射，遍历等），面向CPU，集合关注的是数据的存储，面向内存

Stream API 之于集合，类似于SQL之于数据表的查询

使用说明

1.stream自己不会存储数据

2.stream不会改变源对象，相反会返回一个持有特定结果的新stream

3.stream操作是延迟执行的，意味着他们等到需要结果是再执行，一旦执行终止操作，就执行中间操作链，并产生结果。

4.stream一旦执行终止操作，就不会再调用中间其他操作或终止操作了
