  3. 小结

    1.继承关系:
    HttpServlet -> GenericServlet -> Servlet
    2.Servlet中的核心方法:
    init(), service(), destroy()
    3.服务方法:
    当有请求时，service方法会自动响应（实质是由tomcat容器调用）
    在HttpServlet中会进行判断请求方法，然后再调用请求的方法
    在HttpServlet中这些方法默认都是Http405的实现风格，需要我们自己实现子类，否则会出现405错误
    4.因此，我们在建立Servlet时会考虑请求方法，然后重写该方法

servlet生命周期

  1. 生命周期:从出生到死亡的过程就是生命周期，对应Servlet中的三个方法:init(),service(),destroy()

  2. 默认情况下:
    从第一次接受请求开始，这个servlet对进行实例化（调用构造方法），初始化（调用init()方法），然后服务(调用service()方法)
    从第二次请求开始，每一次都是服务
    当容器关闭时，其中的所有的Servlet实例都会被销毁，调用销毁方法

  3. 通过实例发现
    Servlet实例只会创建一次，所有的请求都是这个实例去响应的

默认情况下，第一次请求时，tomcat才会进行实例化，初始化然后进行服务，这样可以提高系统的启动速度，但是第一次请求时间较长

因此，如果需要提高系统的启动速度，可以选择默认，如果需要提高响应速度，我们应该设置Server的初始化时机

  1. Servlet的初始化时机
    –可以通过在web.xml文件中设置来设置Servlet启动的先后顺序，数字越小启动越靠前，启动越靠前，最小值为0

  2. Servlet在容器中是单例的，是线程不安全的

    单例：所有的请求都是同一个实例进行响应

    线程不安全：一个线程需要根据这个实例的莫各成员变量去进行逻辑判断，但是在中间某个时机，另一个线程改变这个变量，导致另一个线程的执行路径发生了变化

    因此，我们已经知道了servlet是线程不安全的，因此尽量不要在servlet中定义成员变量，如果不得不定义，不要对成员变量的值进行修改同时不要根据成员变量的值进行判断

Http协议

  1. Http被称为超文本传输协议

  2. Http是无状态的（是指协议对于交互性场景没有记忆能力）

  3. Http请求相应包含两个部分 : 请求和响应

    请求
    包含三个部分:1.请求行 ；2.请求消息头 ； 3.请求主体

    1. 请求行包含三个信息:1.请求方式 ；2.请求的URL ；3.请求的协议(一般是HTTP 1.1)

    2. 请求消息头包含了许多客户端需要告诉服务器的信息，比如 : 浏览器的版本，型号，接收内容的类型

    3. 请求体（三种情况）

      get方式，没有请求体，但是有一个queryString

      post方式，有请求体，from data

      json格式，有请求体，request payload

    响应

    包含三个部分：1.响应行 ；2.响应消息头 ；3.响应主体

    1. 响应行包含三个信息：1.协议 ；2.响应状态码（200） ；3.响应状态(ok)
    2. 响应头：包含了服务器的信息，服务器发送给浏览器的信息（内容的媒体类型，内容编码，内容长度等）
    3. 响应体：响应的具体内容

会话

  1. Http是无状态的

    Http无状态：服务器无法判断前后两次请求是同一个客服端发送，还是不同客户端发送

    无状态的现实问题：当第一次请求是添加商品到购物车中，第二次请求结账时服务器无法判断是否是前一个用户的请求

    使用会话跟踪技术解决无状态问题

  2. 会话跟踪技术

    客户端第一次发送请求给服务器，服务器获取session，获取不到，创建新的session,然后响应给客户端

    下一次客户端发送请求时，会把sessionID发送给服务器，服务器就可以获取到请求并进行判断

Thymeleaf

  1. 添加thymeleaf的jar包

  2. 新建一个Servlet类ViewBaseServlet

  3. 在web.xml文件添加配置

    • 配置前缀：view-perfix(/)
    • 配置后缀：view-suffix(.html)

  4. 使得Servlet继承ViewBaseSerlet

  5. 此处的视图名称是index，thymeleaf会将这个逻辑视图名称加到物理视图名称上

    • 逻辑视图名称: index
    • 物理视图名称: view-prefix + 逻辑视图名称 + suffix
    • 真实的视图名称是: /index.html

  6. 使用thymeleaf的标签

    th:if ，th:unless ，th:each ，th:text

保存作用域

原始情况下，保存作用域我们认为有四个：page（页面级别，现在几乎不用），request (一次请求响应范围)，session（一次会话范围） ，application（整个应用程序范围）

  1. request ：一次请求响应范围
  2. session ：一次会话范围有效
  3. application ：一次应用程序范围有效
